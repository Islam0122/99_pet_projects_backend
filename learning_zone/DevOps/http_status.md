# HTTP статусы в веб-разработке

HTTP статус-коды показывают результат обработки запроса сервером.
Они делятся на 5 основных групп.

---

## 1xx — Информационные
Редко используются напрямую в приложениях.

- **100 Continue** — сервер получил заголовки, можно отправлять тело
- **101 Switching Protocols** — смена протокола (например, WebSocket)

---

## 2xx — Успешные

- **200 OK**
  - Запрос выполнен успешно
  - Используется при `GET`, `PUT`, иногда `POST`

- **201 Created**
  - Ресурс успешно создан
  - Часто используется при `POST`
  - Хорошая практика — вернуть созданный объект

- **202 Accepted**
  - Запрос принят, но обработка ещё идёт (асинхронно)

- **204 No Content**
  - Успех, но без тела ответа
  - Часто при `DELETE`

---

## 3xx — Перенаправления

- **301 Moved Permanently**
  - Ресурс перемещён навсегда
  - Важно для SEO

- **302 Found**
  - Временное перенаправление

- **304 Not Modified**
  - Данные не изменились (кеширование)

---

## 4xx — Ошибки клиента

- **400 Bad Request**
  - Неверные данные запроса
  - Часто: ошибки валидации

- **401 Unauthorized**
  - Пользователь не авторизован
  - Нет или неверный токен

- **403 Forbidden**
  - Доступ запрещён
  - Пользователь авторизован, но нет прав

- **404 Not Found**
  - Ресурс не найден

- **405 Method Not Allowed**
  - HTTP-метод не поддерживается (`POST` вместо `GET`)

- **409 Conflict**
  - Конфликт данных (например, дубликат)

- **422 Unprocessable Entity**
  - Ошибка бизнес-логики / валидации
  - Часто используется в REST API

---

## 5xx — Ошибки сервера

- **500 Internal Server Error**
  - Общая ошибка сервера
  - Часто из-за багов в коде

- **502 Bad Gateway**
  - Сервер получил неверный ответ от другого сервиса

- **503 Service Unavailable**
  - Сервер временно недоступен
  - Перегрузка или техработы

- **504 Gateway Timeout**
  - Сервер не дождался ответа от другого сервиса

---

## Практика для API

| Сценарий | Статус |
|--------|--------|
| Успешный GET | 200 |
| Создание объекта | 201 |
| Удаление | 204 |
| Ошибка валидации | 400 / 422 |
| Нет токена | 401 |
| Нет прав | 403 |
| Не найдено | 404 |
| Ошибка сервера | 500 |

---

## Хорошие практики

- Используй **201** при создании ресурсов
- Не возвращай **200** при ошибках
- **422** лучше, чем 400 для валидации
- Всегда возвращай понятное сообщение об ошибке
